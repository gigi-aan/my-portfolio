    Student Violations Management System

    Role

    Front End Developer & UI/UX Designer

    Duration

    August 2024 - March 2025

    Team

    • 1 Project Manager
    • 2 Front End Developers
    • 3 Back End Developers
    • 2 Quality Analysts

    Tech Stack

    HTML, CSS, Javascript, PHP, SQL

    Overview

    As part of our major academic project, I worked on the frontend and UI design of a dashboard that visualizes student violations across different university colleges. The project aimed to help school administrators monitor behavioral trends more efficiently. I focused on creating a clean, user-friendly interface and integrating dynamic charts using Chart.js.

    Problem

    The university lacked a system for monitoring student violations in a consistent and organized way. Most reports were done manually, scattered across departments, and difficult to consolidate. This made it hard to identify recurring issues or take timely action.

    Outcome

    I designed and built the frontend using HTML, CSS, and JavaScript, and connected it with real-time data visualizations. The system was deployed and received positive feedback of the design and user experience during our final defense, highlighting its potential to improve administrative decision-making.
